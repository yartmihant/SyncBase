# I. Состояние проекта

## 1. Сущность проекта

**SyncBase** — консольный инструмент для синхронизации локального древа проектов с Яндекс.Диском.

Проект ориентирован на:

- локальное хранилище произвольных проектов, инициализируемое файлом `.syncbase`;
- двухуровневую структуру хранилища: **категория → проект**;
- безопасную двустороннюю синхронизацию с защитой от случайной перезаписи более новых файлов.

Основная ценность проекта — в согласованности между:

- модулями пакета (`src/syncbase/`);
- тестами (`tests/`);
- документацией (`README.md`, `PROJECT_STRUCTURE.md`).

Проект оформлен как pip-пакет и предназначен для публикации на PyPI.

## 2. Топология проекта

Каноническое описание файловой структуры находится в `PROJECT_STRUCTURE.md`.

При обнаружении различий между фактическим состоянием файлов и содержимым
`PROJECT_STRUCTURE.md` агент обязан сообщить пользователю и, при необходимости,
обновить `PROJECT_STRUCTURE.md`.

## 3. Назначение корневых каталогов

### `src/syncbase/`

Основной код пакета. Модули:

- `cli.py` — точка входа CLI (`syncbase` command), парсинг argv, поиск хранилища.
- `base.py` — класс `SyncBase`: массовые операции, обход категорий/проектов.
- `project.py` — классы `SyncProject` и `SyncIgnore`: синхронизация одного проекта.
- `item.py` — классы `SyncItem` и `ItemState`: состояние одного файла/папки.
- `client.py` — класс `YandexDiskClient`: HTTP-клиент Яндекс.Диска.
- `resolver.py` — функция `find_storage()`: поиск `.syncbase` вверх по дереву папок.

### `tests/`

Интеграционные и unit-тесты. Используют реальное тестовое хранилище (токен в
`tests/test_base/.syncbase`).

### `work/`

Рабочие файлы, планы, заметки.

## 4. Архитектурные правила

- Зависимость `python-dotenv` **удалена**; конфигурация читается из `.syncbase`.
- Файл `.syncbase` содержит строку `YANDEX_DISK_TOKEN=<токен>`.
- Скрипт ищет `.syncbase` от `cwd` вверх до корня ФС; папка с файлом — `BASE_PATH`.
- Хранилище строго двухуровневое: `<base_path>/<category>/<project>/`.
- Флаг `-f` / `--force` к командам `save`/`load` разрешает принудительную перезапись.
- `sys.exit(1)` используется при опасной перезаписи без `--force`.

## 5. Команды разработки

```bash
# Установка в editable-режиме
pip install -e .

# Запуск тестов
pytest tests/

# Проверка типов
mypy src/syncbase/
```

## 6. Версионирование

Версия хранится в `VERSION` и читается через `[tool.setuptools.dynamic]` в `pyproject.toml`.
После значительных изменений обновляйте `VERSION` и добавляйте запись в `CHANGELOG.md`.
